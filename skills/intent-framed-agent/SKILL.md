# Intent Framed Agent スキル

実装開始の瞬間に暗黙の意図を明示的・追跡可能なアーティファクトに変換し、スコープドリフトが可視化された状態で実行を進める。

**スコープ**: コーディングタスク専用。実装作業を伴う場合のみ使用する。
以下は対象外: 研究・調査のみ、計画のみの会話、ドキュメントのみの作業、管理タスク。

`context-surfing` との関係:
- **intent-framed-agent（このスキル）**: スコープドリフト監視（正しいことをやっているか）
- **context-surfing**: コンテキスト品質ドリフト監視（うまくやれているか）
- 両方が同時に発火した場合、context-surfing の退出が優先される（コンテキスト品質低下時はスコープチェックも信頼できないため）

## Phase 1: 意図キャプチャ

実装開始前に以下のフレームを出力し、ユーザーの確認を得る:

```
## Intent Frame #N

**Outcome:** [1文。完了とはどういう状態か]
**Approach:** [実装方法。主要な判断事項]
**Constraints:** [スコープ外の境界]
**Success criteria:** [完了の検証方法]
**Estimated complexity:** [Small / Medium / Large]
```

ユーザーの確認後にコーディングを開始する。

## Phase 2: 実行中モニタリング

自然な区切り（ファイル保存・テスト実行・PR 作成前など）でスコープドリフトを検知する。
ドリフトが疑われる場合:

```
## Intent Check #N

スコープ外の作業に移ろうとしている可能性があります。
[具体的な状況の説明]

これは意図的なピボットですか、それとも偶発的なスコープクリープですか？
```

ユーザーが「意図的なピボット」と回答した場合 → 現在の Intent Frame を更新して続行。
「偶発的」または「戻る」の場合 → 元のスコープに戻る。

## Phase 3: 解決記録

タスク終了時（コミット完了・PR 作成・ユーザーによる終了宣言）に出力する:

```
## Intent Resolution #N

**Outcome:** [Fulfilled / Partially fulfilled / Pivoted / Abandoned]
**What was delivered:** [実際の成果物の概要]
**Pivots:** [承認済みのスコープ変更。なければ None]
**Open items:** [スコープ内で残った未完了タスク。なければ None]
```

## マルチインテントセッション

- 現在のインテントを解決してから次を開始する（並行しない）
- フレームは #1, #2, #3 と順番に番号付けする
- Constraints は明示的に再宣言しない限り引き継がない

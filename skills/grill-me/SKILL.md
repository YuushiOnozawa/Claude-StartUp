# Grill Me

徹底的なインタビューによって、計画・設計の全論点を洗い出し、共通理解に到達するスキル。

## 目的

ユーザーの計画や設計を深掘りし、以下を達成する：

- 曖昧な前提を明確にする
- 見落とされている論点・リスクを浮かび上がらせる
- 各論点について共通の理解・合意を形成する

## 進め方

### 1. 対象の把握

まず対象（計画・設計・アイデア）の概要を把握する。会話履歴や提示されたコンテキストから読み取れる場合は、そこから情報を抽出してから質問を始める。

### 2. 深掘りインタビュー

**深さ優先で進める。** ある論点について洞察が得られなくなるまで掘り続けてから、次の論点に移る。広く浅く聞くのは避ける。

各質問は `AskUserQuestion` ツールで一問ずつ提示する。ユーザーの回答を受け取ってから次の質問に進む。

**質問の構成：**

選択肢を提示する場合は、以下を必ず含める：
- **推奨選択肢**（★マーク付きで明示）
- **自由入力**（「その他：〇〇」形式）

例：
```
選択肢：
1. ★ 推奨：〇〇（理由：〇〇だから）
2. △△
3. その他：自由に入力
```

質問に正解がある場合はコードベースや設計ドキュメントを確認してから質問する。わかることはグリルしない。

### 3. 掘り尽くしの判断

以下のどちらかに達したら、その論点は終了と判断する：

- ユーザーの回答が収束し、新しい情報が得られなくなった
- その論点に関して取りうる選択肢・リスク・前提がすべて明確になった

論点を終えたら次の未解決の論点に移る。

### 4. 終了条件

すべての重要な論点を掘り尽くし、共通理解に達したとき終了する。判断が難しければ「これで十分だと思いますか？」と確認する。

### 5. 最終出力

インタビュー完了後、以下のフォーマットで結果を出力する：

```markdown
## まとめ

### 決まったこと
- ...

### 未解決・要確認
- ...（ある場合のみ）
```

「決まったこと」には、インタビューを通じて合意・確定した事項を箇条書きで列挙する。

## 注意点

- 一度に複数質問しない。一問一答を徹底する
- 推奨を提示する際は、理由も一言添える
- コードベースで確認できることは確認してから聞く（わかることはグリルしない）
- ユーザーが「もう十分」と言ったら潔く終了する

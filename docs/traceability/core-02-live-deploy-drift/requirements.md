# Requirements: Core 02 — 実働環境で生まれた開発内容の還流経路が未定義

> 旧番号: Core 05。2026-07-07 の前提変更により全面改訂（status: draft・再承認待ち）。
> 旧版の問題定義「本番 ~/.claude とリポジトリの正が分裂している」および旧 REQ-02-01〜08 は破棄。
> 経緯は `opus-context-2026-07-07.md`（前提変更の発覚）と `sonnet-handoff-2026-07-07.md`（決定と再定義）を参照。

## 用語（2026-07-07 確定）

| 用語 | 意味 |
|---|---|
| **実働環境** | `~/.claude/`。開発最適化された作業環境。リポジトリと一致している必要はない。旧称「本番」は前提変更前の世界観（リポジトリが正・~/.claude はその配備先）を含意するため**廃止** |
| **配布原本** | 開発リポジトリ `~/srcs/Claude-StartUp`。「新規環境で同一環境を構築する」ための正。実働環境の上位ではなく、還流の**終着点** |
| **還流** | 実働環境で生まれた開発内容を配布原本へ PR として戻すこと |
| **内容物** | 新規環境でも再現されるべきファイル群（skills/hooks/agents 等）。ローカルデータ（セッション・ログ・マシン固有設定）と区別する |

## 背景

リポジトリの根本目的は「新規マシン（WSL2）で同一の Claude Code 環境を自動セットアップ・完遂できること」。
2026-07-07 のユーザー確認により、実働環境 `~/.claude/` が配布原本と diverge していること自体は問題ではなく、
実働環境は開発作業のために最適化されていればよい、と前提が変更された（Fable 15 監査の暗黙前提
「~/.claude はリポジトリの正コピーであるべき」を破棄）。

前提変更後も残る本質的な問題は、**開発内容が実働環境で生まれる**（実例: traceability-*, lean-ctx,
compact-prep 等のスキル群が実働環境にのみ存在し未マージ）にもかかわらず、**実働環境 → 配布原本の
還流経路が未定義**であること。還流されない開発内容は、配布原本の網羅性を静かに劣化させ、
根本目的（新規環境での同一環境構築）を直接損なう。

方向性の確定（2026-07-07）: 還流ルート（実働環境 → 配布原本）の確立が**必須**、
配備方向（配布原本 → 実働環境）のツールは**任意**の利便機能とする。
配備方向を完璧にしても実働環境で生まれた変更は救えないため。

関連Fable項目: 07（一部は core-03.4 へ移管）, 15

## 問題

| ID | 問題 |
|---|---|
| PROB-02-01 | 実働環境で生まれた開発内容（新スキル・hooks 等）が、配布原本に還流されないまま実働環境にのみ存在し得る。還流漏れは新規環境再現性を静かに劣化させる（実例: traceability-*, lean-ctx, compact-prep 等が未マージ） |
| PROB-02-02 | 還流漏れを機械的に検知する仕組みがなく、「戻し忘れ」の防止が運用の善意に依存している |
| PROB-02-03 | 両側変更ファイル（`MM`: CLAUDE.md, skills/pr-review/SKILL.md）の突合が未了で、実働環境側の有益な変更が失われ得る |
| PROB-02-04 | 実働環境が約76PR遅れの git clone のまま残っており、git 状態（dirty 30+）が実態を説明できない。誤って `git pull` すると大量コンフリクト事故が確定している |

> 旧 PROB-02-04（worktree 残骸）はリポジトリ衛生の問題であり本核問題の範囲外 → **core-03.4 へ移管**

## 確定した要求

| # | 要求 | 種別 | 根拠 |
|---|---|---|---|
| REQ-02-01 | `~/.claude` を**通常ディレクトリ**とし、git 管理から外す（de-git）。理由は「無意味化した git 計器の撤去 + pull 事故防止」（旧理由「配備方式の前提」は破棄） | 必須 | 2026-07-07 確定（前提変更後に再確認） |
| REQ-02-02 | **内容物ホワイトリスト**を明示的に定義する: 実働環境のうち何が「配布原本に収載されるべき内容物」で、何が「ローカルデータ」かの区分（旧 deploy.sh ホワイトリストを意味を変えて転用） | 必須 | 2026-07-07 確定 |
| REQ-02-03 | **還流検知**の手段を設ける: ホワイトリスト対象について実働環境と配布原本を突合し、実働環境にのみ存在する内容物・差分を一覧化するスクリプトまたはスキル | 必須 | 2026-07-07 確定（(b) 必須の帰結） |
| REQ-02-04 | **還流手順**を定型化する: 検知結果を PR 化する定型ルートを定義し、README に明記する（旧「直編集したら速やかに PR」の注意書きを、検知→PR の運用手順へ昇格） | 必須 | 2026-07-07 確定 |
| REQ-02-05 | `settings.json` は還流・配備の対象外とする。setup スクリプト（410/411/412/700/250）が冪等に管理する | 必須 | 2026-07-07 確定（前提変更の影響なし・維持） |
| REQ-02-06 | ローカル専用ファイル（`CLAUDE.local.md`、`settings.local.json`、`projects/`、`paste-cache/`、`hooks/logs/`、`hooks/queue/`、`tasks/` 等）は還流・配備の対象外とし、de-git 後もそのまま残す | 必須 | 2026-07-07 確定（維持） |
| REQ-02-07 | 配備方向ツール（配布原本 → 実働環境。rsync ホワイトリスト方式等）は**任意**の利便機能として提供してよい。確実性・網羅性は要求しない。REQ-02-02 と同一のホワイトリストを使う | 任意 | 2026-07-07 確定（deploy.sh を必須経路から格下げ） |
| REQ-02-08 | 還流検知（REQ-02-03）は**手動スキルとして提供**する。利用側（hooks/フロー）への影響を出さない形とする。起動は開発者の判断に委ねる | 必須 | 2026-07-07 確定 |

## 内容物ホワイトリスト（要求レベルの方針）

**内容物**（還流検知の対象 = 配布原本に収載されるべきもの）:
- `skills/`、`hooks/`（スクリプト類）、`agents/`、`dotfiles/`、`scripts/`、`rules/`、`commands/`、`templates/`、`CLAUDE.md`

**ローカルデータ**（還流・配備とも対象外）:
- `settings.json`（setup スクリプトが管理）
- `projects/`、`paste-cache/`、`hooks/logs/`、`hooks/queue/`、`tasks/`（セッション・ランタイムデータ）
- `CLAUDE.local.md`、`settings.local.json`（マシン固有）
- `*.bak`（バックアップファイル）

> 詳細な確定は Step 4（specification）で行う

## 受け入れ条件

- `~/.claude` に `.git/` ディレクトリが存在しない
- 内容物とローカルデータの区分が文書化されている
- 還流検知を実行すると、実働環境にのみ存在する内容物・差分の一覧が得られる（実例で確認）
- 検知 → PR 化の手順が README に明記され、起動タイミング（REQ-02-08 で確定した方式）から辿れる
- 初回の還流棚卸しが実施され、`MM` ファイル（CLAUDE.md、skills/pr-review/SKILL.md）の取捨判断が記録されている
- `settings.json` とローカル専用ファイルが対象外であることが明記され、（配備ツールを実装する場合）配備実行後も変化しない

## 対象外

- de-git 実行の具体的な手順・スクリプト実装（Step 5〜6）
- 還流検知・配備ツールの実装（Step 5〜6）
- `agents/leliel.md` の削除 → **core-03.1** で対処（還流検知はこれを「実働環境にのみ存在」と検出するが、削除予定物として還流しない）
- **worktree 残骸・`.gitignore` 整合（旧 Fable 07 由来）→ core-03.4 へ移管**
- setup 完遂保証（ワンライナー展開後の実行可能状態）→ core-03.3
- settings.json の内容整合 → setup スクリプト群（core-03.3）の範囲
- pCloud への一括転送設計 → core-01 REQ-01-04〜06 / core-03.2 参照

- core-03.1（agents/leliel.md 削除）→ ハードブロックではなくなった。還流検知の「既知の削除予定物」として記録すれば還流誤爆を防げる

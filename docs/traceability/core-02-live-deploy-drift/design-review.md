## 推奨修正

### 推奨（ブロッカーではない）: SPEC-02-03 への補足追記

下記の 2 点を SPEC-02-03 の「処理アルゴリズム」または「境界条件」に追記することを推奨する:

1. **比較対象の確定**: `sync-whitelist.conf` の `+ ` include 行のみを読む。`- ` exclude 行はスクリプトが参照しない（exclude 行は rsync 配備ツール（SPEC-02-06）用の文書として存在する）
2. **比較方向**: 実働環境にのみ存在するファイルを「要還流（新規）」として出力する。配布原本にのみ存在するファイルは出力しない（還流は実働環境→配布原本の一方向）

### 推奨（ブロッカーではない）: 終了コードの明記

SPEC-02-03 の fail/warn/info 表に終了コードを追加:

| 状態 | 判定 | 終了コード |
|---|---|---|
| 設定ファイル不存在・パス不存在 | fail（中断） | exit 2 |
| 要還流（新規）または（変更）が1件以上 | warn | exit 1 |
| 要還流 0件 | success | exit 0 |

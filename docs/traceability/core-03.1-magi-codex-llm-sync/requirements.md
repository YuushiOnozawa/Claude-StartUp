# Requirements Draft: Core 02

## 背景

MAGI、ローカルLLM、Codex の実体が、参照パス、モデル割当、ドキュメント、setup で同期していない。開発フローのレビューゲートが期待通り動くかに直結する。

関連Fable項目: 01, 02, 03, 06, 08

## 問題

この核問題では、分類成果物で整理された問題を要求定義へ進める前段階として扱う。ここに書く内容は要求候補であり、確定要求ではない。

## 要求候補

- MAGIペルソナ定義の正を agents か references のどちらかに固定する
- スキル要求モデル、setup pullモデル、モデル表を一致させる
- グローバルスキルが任意cwdから動作することを要求する
- Codex実装・監査・設計レビュー層が利用可能であることを要求する

## 受け入れ条件候補

- リポジトリ外gitプロジェクトで /magi-fast / /magi-hard が完走するか
- bash scripts/、固定codexパス、存在しないagents参照をCIで検出するか
- スキル側モデルとsetup側モデルを突合できるか
- Codex CLI未導入時の扱いを検出できるか

## 対象外候補

- コード変更・実装修正そのものは、この整理ステップでは対象外。
- 関連ドキュメントの最終文言確定は、要求確定後の別作業とする。
- 他の核問題で主担当となる項目は、このフォルダでは重複関係として扱う。

## 人間確認事項

- #203 の方針を完遂して references に一本化するか agents を復元するか
- METATRON の devstral を継続するか
- Codex CLI / plugin の確認済みバージョンをどこまで固定するか
